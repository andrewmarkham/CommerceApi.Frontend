# Jhoose Commerce Framework

## What is it

The Jhoose Commerce Framework extends Optimizely Commerce Connect / AKA B2C Commmerce.

#### Customer Rest API
A Rest API layer that supports these common eCommerce journies

See [Documentation](https://github.com/andrewmarkham/CommerceApi/)


#### Javascript/Typescript Customer API Layer
This library provides an abstration of the Rest API which simplifies the approach when developing either clientside components in Vanilla JS or more complex applications via Frameworks such as NEXT.js or Angular.
- Parity functionality to the Customer Rest API
- Caching to enhance performance

#### Microfront UI Layer
This is a React framework that delivers a core set of components that can be used to accelerate development.

- Add to Basket
- Product Price
- Mini Cart
- Market Selector
- Checkout

The components can be styled to match designs.

---
## Why

#### Time to market.
Optimizely - Commerce Connect is a flexible and scaleable solution, but it takes effort to deliver.

The aim of the Jhoose Commerce framework is to shorten the time to market.

- The Customer Rest API - Provides access to managing a cart, the customer and checkout.
- The Javascript/Typescript Customer API - Simplifies using the Rest API when building the frontent.
- The Microfront End - React components that can be used in any

#### Support more modern architectural approaches.
Organisations are considering headless eCommerce solutions. The Jhoose Commerce framework provides both the API layer and also a Javascript/Typescript framework allowing organisations to use Optimizely Commerce Connect as a headless eCommerce system without any addional development.

#### Support multiple channels.
The Rest API can be used by Websites, POS, Mobile apps, surfacing the same Cart, Customer and Checkout flows.
